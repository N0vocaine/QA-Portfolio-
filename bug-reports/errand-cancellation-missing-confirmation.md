# Bug Report: Missing confirmation message after cancelling errand

## Title
No confirmation message displayed after cancelling an errand in queue

## Environment
- Platform: Android
- Device: Samsung Galaxy S21 5G
- App version: 3.63.0

## Preconditions
- User is logged in
- User has an active subscription 
- User has an ongoing errand placed in queue

## Steps to Reproduce
1. Create an errand in the "Doctor" category
2. Wait until the errand is placed in the queue
3. Cancel the errand

## Actual Result
The user is redirected to the home screen without any confirmation message indicating that the errand has been closed.

## Expected Result
A confirmation message should be displayed to the user after cancelling the errand, for example:
“Your conversation is now closed. You are always welcome to start a new conversation.”

## Impact
- User may be unsure if the errand was successfully cancelled
- Creates confusion and lack of feedback
- Reduces trust in the system

## Severity
Medium
