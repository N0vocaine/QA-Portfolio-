# Bug Report: "Save" button overlaps text in Permissions screen

## Title
"Save" button overlaps text in Permissions screen during onboarding

## Environment
- Platform: Android
- App type: Mobile application

## Preconditions
- User is in the onboarding flow
- User reaches the Validation → Permissions step

## Steps to Reproduce
1. Start onboarding flow
2. Proceed to the Validation → Permissions step
3. Observe the layout of the screen

## Actual Result
The "Save" button overlaps the text, making part of the content difficult to read.

## Expected Result
The "Save" button should be properly positioned and should not overlap any text. All content should be clearly visible.

## Impact
- Reduces readability
- Affects user experience during onboarding
- Creates UI/layout inconsistency

## Severity
Low to Medium
